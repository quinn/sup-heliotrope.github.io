Hi,

I think this is a step in the right direction - given that there has 
been silence on wmorgans part so long.

Kepping things in one place does work alot better, even if it means 
sacrificing a few features - I think that keeping the wiki along with 
the rest of the organization is the best.  Also github is not linked or 
dependant on any one admin / developer, so I think for this kind of 
project it is ideal - expanding to a proper web site is no biggie with 
github either, making future possible migration even easier.

I'd like to be part of the organization, but I am probably not going to 
be active enough to be an administrator, github name: gauteh.

Cheers, Gaute
