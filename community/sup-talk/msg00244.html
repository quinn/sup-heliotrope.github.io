Glad that helped. It was a bit of a code-dump because I'm not using
sup any more.

The abandon-ness of sup is sad; it was probably the best mail client
I've used (except for mh, about 20 years ago). Unfortunately as my
underlying OS libraries changed, sup didn't keep up with API changes,
and now it can't handle GPG. For a while I was running a chroot Ubuntu
11.04 just for sup, but I stopped doing that a while ago. As others
have mentioned, nwcurses is the solution to utf8 but applying it isn't
straightforward. And I didn't ever try multiple personalities (as I
keep my personalities in separate areas/servers and try hard not to
mix them in the same UI; too much chance of data leakage otherwise and
that's not a good thing in my line of work)

I'm currently trying to keep heliotrope/turnsole running, but it seems
to be only halfway useable; the basics work but the more advanced
things are not present. I'm not a developer and can't wade in to @wm's
code to help there. It does keep my daily mailbox running though.

I'd go back to mh again, possibly combined with some tmux invocations
to keep the asynchronous nature, except I've grown to like the use of
a tagging interface rather than the traditional Maildir/IMAP folder
system. However, notmuchfs is a tempting general-purpose solution to
that.
